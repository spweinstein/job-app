## 2026-05-25 — Use @supabase/ssr for Supabase client utilities

**Branch:** claude/scaffold-00a-plan-iGKPE
**Context:** The spec deliverables list `src/lib/supabase/server.ts`, `src/lib/supabase/client.ts`, and `src/lib/supabase/middleware.ts`. The modern Supabase library for Next.js App Router cookie-based auth is `@supabase/ssr`.
**Decision:** Use `@supabase/ssr` (alongside `@supabase/supabase-js`) to implement the three Supabase client helpers. This is Supabase's official recommendation for App Router.
**Consequence:** `@supabase/ssr` must be added as a dependency. The three helper files export `createServerClient`, `createBrowserClient`, and `updateSession` respectively.

## 2026-05-25 — Stub src/types/database.ts for Phase 0

**Branch:** claude/scaffold-00a-plan-iGKPE
**Context:** The spec says `src/types/database.ts` is "Generated by Supabase CLI" after every migration. Phase 0 has no migrations, so the CLI cannot generate types yet.
**Decision:** Create a stub `database.ts` with `export type Database = Record<string, never>` that satisfies TypeScript strict checks and will be replaced by `supabase gen types typescript --local` in Phase 1 once the first migration exists.
**Consequence:** TypeScript will compile cleanly in Phase 0. The stub must be replaced (not merged) when Phase 1 runs type generation.

## 2026-05-25 — Read observability.md despite it being absent from the Phase 0 reading list

**Branch:** claude/scaffold-00a-plan-iGKPE
**Context:** The roadmap Phase 0 reading list names only `docs/technical-spec/index.md` and `docs/technical-spec/testing.md`, but Phase 0 deliverables include Sentry initialization which requires the observability spec. This was flagged as a gap by the discovery phase.
**Decision:** Read `docs/technical-spec/observability.md` when planning and building Phase 0 to ensure spec-compliant Sentry and logger implementation.
**Consequence:** Logger fields (`level`, `message`, `timestamp`, `requestId`, `userId`, `action`, `durationMs`, `error`) and Sentry init placement (`src/instrumentation.ts` + `src/app/layout.tsx`) are correctly specified in the Phase 0 prompt.
